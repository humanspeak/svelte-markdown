## Findings considered and rejected

- **Mid-document-edit token identity reuse**: `canReuse` is gated to
  append-only updates (`incremental-parser.ts:564`), so editor-style edits and
  offset-mode rewrites discard all previous token objects even when most of
  the document is byte-identical. Source-offset keys already prevent DOM
  destroy/recreate for the unchanged prefix; the remaining cost is prop-level
  re-render/diff. Deferred (not planned): L effort, HIGH risk, and it only
  pays off for editor-style consumers, which are not a confirmed audience.
  Revisit on demand.
- **Source-less root-key simplification** (pre-parsed token-array inputs fall
  back to object-identity keys): already investigated and REJECTED as plan 012
  of the closed `streaming-component-hardening` batch — both candidate
  simplifications regressed first-child churn. Do not re-audit.
- **Reference-definition sensitivity cost**: verified benign — an append that
  introduces a reference definition costs one full-redraw flush, not every
  flush (`incremental-parser.ts:559-560`). No action.
